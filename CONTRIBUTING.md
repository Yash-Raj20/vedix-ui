# Contributing to Vedix UI

Thank you for your interest in contributing to **Vedix UI**! 🎉
We welcome contributions from developers of all experience levels.

Vedix UI aims to be a modern, developer-first React UI component system. Contributions such as bug fixes, new components, documentation improvements, and feature suggestions are all appreciated.

---

## Ways to Contribute

You can contribute in several ways:

* 🐛 Reporting bugs
* ✨ Suggesting new features
* 🧩 Adding new UI components
* 📚 Improving documentation
* 🛠 Fixing bugs or improving existing components
* 💡 Improving CLI functionality

---

## Getting Started

### 1. Fork the Repository

Fork the repository to your GitHub account and clone it locally.

```bash
git clone https://github.com/Yash-Raj20/vedix-ui.git
```

Navigate into the project directory:

```bash
cd vedix-ui
```

---

### 2. Install Dependencies

Install project dependencies using your preferred package manager.

```bash
npm install
```

---

### 3. Create a Branch

Create a new branch for your contribution.

```bash
git checkout -b feature/your-feature-name
```

Examples:

```
feature/add-slider-component
fix/button-hover-style
docs/update-installation-guide
```

---

## Development Guidelines

Please follow these guidelines when contributing:

### Code Style

* Use **TypeScript** where possible
* Follow existing project structure
* Keep components modular and reusable
* Use **Tailwind CSS** for styling
* Ensure accessibility best practices

### Component Structure

Each component should include:

* Clean and reusable code
* Proper TypeScript types
* Accessible markup
* Minimal dependencies

Example structure:

```
src/components/ui/component-name
```

---

## Adding a New Component

If you want to add a new component:

1. Create the component inside the `components/ui` directory
2. Follow existing component patterns
3. Ensure the component is reusable
4. Add example usage in the documentation site

---

## Commit Guidelines

Use clear and descriptive commit messages.

Examples:

```
feat: add slider component
fix: improve button focus state
docs: update installation section
```

---

## Submitting a Pull Request

When your changes are ready:

1. Commit your changes

```
git add .
git commit -m "feat: add new component"
```

2. Push your branch

```
git push origin feature/your-feature-name
```

3. Open a Pull Request on GitHub

Please include:

* A clear description of the change
* Screenshots (if UI related)
* Steps to test the change

---

## Reporting Bugs

If you find a bug, please open an issue and include:

* Description of the issue
* Steps to reproduce
* Expected behavior
* Screenshots if applicable

---

## Feature Requests

We welcome feature suggestions!
Please open an issue and describe:

* The problem you are trying to solve
* Your proposed solution
* Any alternatives considered

---

## Code of Conduct

Please be respectful and constructive when interacting with others.
We aim to maintain a friendly and welcoming community.

---

## Questions

If you have any questions, feel free to open a discussion or issue.
